# Conventions

- Each feature uses bash strict mode: `#!/usr/bin/env bash` and `set -euo pipefail`.
- Each install script is written as an entrypoint that runs as root.
- Scripts use the same section order and headings:
   1. Inputs
   2. Collect options
   3. Logging functions
   4. Check option compatibility
   5. Prerequisites
   6. Check required commands
   7. Installation
   8. Install functions
   9. Install packages
- Shell variables use lowercase `snake_case`; Feature option environment variables are read once into normalized local variables.
- Scripts use the same `log`, `error`, and `require_command` helpers. Installers that invoke apt directly use the shared `install_apt_packages` helper.
- `install_apt_packages` performs one complete apt cycle: `apt-get update`, `apt-get install -y --no-install-recommends`, `apt-get clean`, and removal of `/var/lib/apt/lists/*`.
- Every declared option changes installer behavior. Options that a Feature cannot implement, such as conditional host bind mounts, belong in the consuming `devcontainer.json` instead.
- npm installations use `--global --ignore-scripts --min-release-age`; constrained pipx installations use pip's `--uploaded-prior-to` argument. Both use a `minReleaseAge` option that defaults to seven days.
- Features fail early with a clear message when an optional selected capability requires a missing prerequisite, such as npm.
- Each feature exposes options through `devcontainer-feature.json` and consumes them through uppercase environment variables in `install.sh`.
- `installsAfter` is reserved for portable, published-feature dependencies. Local test scenarios use `overrideFeatureInstallOrder` when they compose this repository's local Node feature before npm-dependent features.
- Feature options describe only behavior that the Feature specification can implement. Host mounts and user configuration remain the responsibility of the consuming `devcontainer.json`.
- Tests use `ubuntu:latest` for scenario and integration coverage. Node-dependent scenarios compose the local Node feature explicitly; autogenerated isolated tests retain a Node-equipped base image where the Dev Container CLI cannot express that composition.
